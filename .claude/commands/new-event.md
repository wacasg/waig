---

`index.html` の `TURN_POOLS` に新しいイベント候補を1つ追加する。$ARGUMENTS の指定（スロットと狙い）に従う。

## イベントの型（既存に厳密にそろえる）
```js
{
  tag:"今月の出来事 ①",        // スロット番号 ①導入 / ②中盤 / ③ラスト。バランス計算もこのタグで分類する
  title:"<短いキャッチ>",       // 高校生が一目で状況を掴める
  desc:"<状況説明1〜2文>",
  q:"店長、どうする？",          // スロットの問いに合わせる
  aiRec:"A",                    // セオリー通りの“無難な”手。必ずしも最適でないのが肝
  ai:"<AIの一言。断定しすぎず根拠を添える>。Aがおすすめ。",
  actions:[
    {id:"A", icon:"📣", t:"<集客寄りの手>",     d:"<ねらい>", eff:{kya:18}},
    {id:"B", icon:"🛠️", t:"<使いやすさ寄りの手>", d:"<ねらい>", eff:{tsu:22,kai:6}},
    {id:"C", icon:"🏷️", t:"<買いたい度寄りの手>", d:"<ねらい>", eff:{kai:16,kya:2}}
  ]
}
```

## ルール
- **そのスロット内の既存3候補と効果量(`eff`)をそろえる**（合計値・主メーターの伸びを同水準に）。引きで難易度がぶれないこと。
- A/B/C はそれぞれ別メーター（集客 `kya` / 使いやすさ `tsu` / 買いたい度 `kai`）を主に伸ばす。
- 文言は高校生向け・専門用語ゼロ（`copy-edu-reviewer` の基準）。`tag` のスロット番号を間違えない。
- 既存の絵文字・口調・体裁に合わせる。

## 追加後（必須）
1. `python3 scripts/balance_sim.py --seed 1` で追加前後のクリア率を比較し、難易度が大きく動いていないか確認。
2. `/preview` で実プレイし、新イベントが正しく抽選・表示・計算されるか確認。
