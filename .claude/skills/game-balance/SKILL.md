# WAIG ゲームバランス調整スキル

`index.html` の数値（`target` / 各 `eff` / 開始市場レンジ）を**体感ではなくシミュレーション**で調整するための手順。

## シミュレータの仕組み
`scripts/balance_sim.py` は `index.html` を直接パースして
`calcYen` の係数・`CONFIG.target`・全イベントの `eff`・`aiRec`・開始市場レンジを読み取り、
モンテカルロでプレイを再現する。**数値の二重管理は無い**（index.html を直せば結果も追従）。

戦略別に評価する:
- **最適（理論上限）** … その回の引きで取れる最大売上（3^3 全パス探索）。考え抜いた子の上限。
- **弱点補強** … 毎ターン最小メーターを最も伸ばす手。賢いプレイ。
- **AI全追従** … `aiRec` を毎回選ぶ。鵜呑みプレイ。
- **ランダム** … 適当に選ぶ。下限の目安。

## 手順
1. **現状把握**: `python3 scripts/balance_sim.py --games 20000 --seed 1`
   - 戦略別クリア率・平均売上・ランク分布(S/A/B/C)が出る。`--seed` 固定で前後比較が安定する。
2. **目安と照合**（CLAUDE.md §3）:
   - AI全追従: **45–70%**（簡単すぎず、鵜呑みだと時々落ちる）
   - 最適: **85%+**（考えた子が報われる）
   - ランクが S/C に極端に偏らない
3. **変更**: ズレている原因に応じて `index.html` を編集する。
   - 全体が難しすぎ/簡単すぎ → `CONFIG.target` を動かす（最も影響が素直）。
   - 特定戦略だけ歪 → 該当スロットの `eff` を微調整（**3候補の効果量はそろえる**）。
4. **再検証**: 同じ `--seed` でもう一度実行し、**変更前後を並べて**報告。副作用（他戦略・ランク分布）も確認。
5. **不変条件チェック**: 1ゲーム3ターン／各スロット3候補の役割・効果量が揃っているか。崩れていれば戻す。

## 報告フォーマット
```
変更点: <何をどう変えた>
        target 520000 → 480000  /  slot① B eff tsu:22→20 など
前後比較（--seed 1, 20000回）:
  戦略        クリア率(前→後)   平均売上(前→後)
  最適        66.8% → XX.X%     ...
  AI全追従    20.4% → XX.X%     ...
判定: 目安レンジ内か / 副作用の有無
```

## 注意
- `eff` のキーは `kya`(集客) / `tsu`(使いやすさ) / `kai`(買いたい度)。clamp で 0–100。
- パースに失敗するとスクリプトがエラーを出す（index.html の書式を大きく変えたサイン）。その時は `balance_sim.py` の正規表現を追随させる。
- 数値を変えたら必ずローカルで実プレイ確認（`/preview`）してから push。
